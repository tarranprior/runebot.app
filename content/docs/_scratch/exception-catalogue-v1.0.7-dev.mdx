## Search Tools — Exception Cases (Developer Reference)

### `Nonexistence`

- **Cause**: A Search Tools parser could not match the provided search query to an Old School RuneScape term or wiki/article result.

- **User Message**:

> The term you have searched for does not exist in Old School RuneScape. It may have never existed, or was added to RuneScape after the August 2007 Archive of RuneScape, which Old School RuneScape was based on, and has not been replicated.
>
> If you are searching for a future update, please look for it at [upcoming updates](https://oldschool.runescape.wiki/w/Upcoming_updates) or create an article about it, supplying definite sources and citations.

- **Preview**:

- **Visibility**: Public/non-ephemeral.
- **Status**: Confirmed from current live Discord captures across Search Tools and bot source.

## `/stats` — Exception Cases (Developer Reference)

### `UsernameNonexistent`

- **Cause**: `/stats` was run without a username and the command author does not have a saved/default RuneScape account.

- **User Message**:

> You don't have any accounts yet. Use `/setrsn` to add your first account, or add the `username` argument to `/stats`.
>
> **Usage**: `/stats` or `/stats <USERNAME> [ACCOUNT_TYPE (optional)]`

- **Preview**:

<DocsErrorPreview id="error-no-saved-accounts" ephemeral className="mt-6" />

- **Visibility**: Ephemeral.
- **Status**: Confirmed from current live Discord capture and bot source.

---

### `MentionedUserAccountNonexistent`

- **Cause**: `/stats` was run with a Discord user mention, but the mentioned Discord user does not have a saved/default RuneScape account.

- **User Message**:

> The user you mentioned does not have a default RuneScape account set. Ask them to use `/setrsn` to set a default account, or search with a RuneScape username.

- **Preview**:

<DocsErrorPreview id="mentioned-user-account-nonexistent" ephemeral className="mt-6" />

- **Visibility**: Ephemeral.
- **Status**: Confirmed from current live Discord capture and bot source.

### `NoGameModeData`

- **Cause**: `/stats` was run with a valid username, but the selected `account_type` does not match the account found on the Hiscores.

- **User Message**:

> The username provided doesn't appear to exist under this `account_type`. Please select a different `account_type`, or try another username.
>
> **Usage**: `/stats` or `/stats <USERNAME> [ACCOUNT_TYPE (optional)]`

- **Preview**:

<DocsErrorPreview id="stats-no-gamemode-data" className="mt-6" />

- **Visibility**: Public/non-ephemeral for direct `/stats` lookup. Component refresh/navigation paths edit the original stats response.
- **Status**: Confirmed from current live Discord capture and bot source.
- **Known Discrepancies**:
    - The Usage line may still be simplified or moved into the main `/stats` usage docs before public troubleshooting docs are finalised.

---

### `WikiRequestFailed`

- **Cause**: Runebot could not reach an upstream Old School RuneScape Wiki/API endpoint because of rate limiting, blocking, or temporary availability issues.

- **User Message**:

> The Old School RuneScape Wiki could not be reached for this request. This may be due to rate limiting, temporary blocking, or an upstream availability issue. Please try again shortly.

- **Preview**:

<DocsErrorPreview id="error-wiki-request-failed" className="mt-6" />

- **Visibility**: Public/non-ephemeral when surfaced through the normal `/stats` expected-failure handler.
- **Status**: Code-backed from current bot source. Live capture is not required for this `/stats` catalogue pass because this path depends on upstream blocking/rate-limit conditions that are difficult to trigger reliably.
- **Known Discrepancies**:
    - The message specifically references the Old School RuneScape Wiki. If this exception is surfaced from a Hiscores/API path, the wording may confuse users and should be reviewed during exception copy polish.

## Shared Acknowledgements — Interaction / Runtime (Developer Reference)

These entries are shared acknowledgement paths used across commands or component interactions. They are listed separately so command-specific exception sections stay focused on their own user flows.

### `wrong_component_user`

- **Cause**: A user other than the original command author clicked a component tied to another user's interaction.

- **User Message**:

> Only the original author can use these buttons.

- **Preview**:

<DocsErrorPreview id="wrong-component-user" ephemeral className="mt-6" />

- **Visibility**: Ephemeral.
- **Status**: Code-backed from current bot source.
- **Implementation Note**: Owner-mismatch component acknowledgements are now handled by the shared `ack_wrong_component_user(...)` helper.

---

### `runtime_failure`

- **Cause**: An unexpected runtime error occurred while Runebot was handling a command or component interaction.

- **User Message**:

> Something went wrong while handling that request. Please try again.

- **Preview**:

<DocsErrorPreview id="runtime-failure" ephemeral className="mt-6" />

- **Visibility**: Ephemeral.
- **Status**: Code-backed from current bot source.
- **Implementation Note**: Runtime failure acknowledgements now use the shared `ack_runtime_failure(...)` helper. Existing error-level logging remains in each command handler; the helper only sends the neutral user-facing embed.
